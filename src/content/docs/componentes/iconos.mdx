### ¿Cómo se utiliza?

Los SVG están definidos de forma global en la aplicación, por lo cual solo basta implementarlo directamente en el archivo página `.aspx` o control de usuario `.acsx`. Para ello solo basta con declarar la siguiente línea de código.
```
<componentes:Iconos ID="IconoId" runat="server" />
```
> Si se requiere acceder netamente desde code-behind debes declarar obligatoriamente `runat="server"` y su ID único.

### Propiedades

|Propiedad| Descripción | Tipo de Dato | Valor Predeterminado |
|--|--|--|--|
|**Tipo**|Nombre del icono| `String (Enum)` | LogoDomus |
|**FillColor**| Color de relleno | `String` | currentColor |
|**Size**| Tamaño del icono en X e Y | `Integer` | 16 |
|**StrokeWidth**| Tamaño borde interior del icono | `Integer`| 1 |
|**StrokeColor**| Color del borde interior del svg | `String`| currentColor |

### Ejemplos

```
<telerik:RadButton 
  ID="btnEliminar" 
  runat="server" 
  RenderMode="Lightweight" 
  Primary="true" 
  OnClick="btnElimninar_Click">
    <ContentTemplate>
        <componentes:Iconos ID="iEliminar" runat="server" Tipo="Eliminar" Size="16" FillColor="#fff" />
        <span>Imprimir</span>
    </ContentTemplate>
</telerik:RadButton>
```

### ¿Cómo agregar un nuevo SVG?
Para añadir un nuevo SVG a la galería de iconos del proyecto, accederemos a una ruta del proyecto y una rama del repositorio remoto.

> 💡 Para obtener el icono puedes acceder a los siguientes enlaces:
> 1. **[IconFinder](https://www.iconfinder.com/):** Seleccionar especialmente iconos Outline y dentro del paquete `Free`
> 2. **[Lucide](https://lucide.dev/):** Librería de iconos OpenSource
>
> *Considera colocar por defecto en el SVG un Width y un Height en **32**, con finalidad que se pueda acoplar en la documentación*

- **Proyecto Domus**

  Redirigete hacia el directorio `DD_DomusData/Content/IconosSVG.acsx.vb`, una vez dentro agrega el nombre de tu icono dentro del Enum *TipoIconos*, asegurate que el nombre se encuentre en idioma español.

  Posteriormente, abre tu SVG en el IDE Visual Studio Community/Visual Studio Code o desde la [web](https://vscode.dev/). Selecciona o busca todas las comillas dobles `"` y reemplazalas por `""`. Es decir si el svg contiene `width="32"` debería quedar como resultado `width=""32""`.

  > También ten en cuenta que tu svg no contenga etiquetas `<style>` ya que podría alterar el consumo de los estilos asignados al componente.

- **Branch DOCS**

  Si agregas un nuevo SVG al componente en el proyecto DomusData, también debes agregar físicamente tu SVG al proyecto de documentación.
  
  Para ello deberas cambiarte de rama en tu equipo, por medio de los siguientes comandos:

  - `git fetch origin docs`: Para traer los últimos cambios emitidos a la rama de documentación a tu repositorio local.
  - `git checkout docs`: Git realizará el cambio de rama sin llevar cambios de una rama anterior. 
    > Si la rama no se encuentra creada en local, Git creará automáticamente una rama local que rastrea esa rama remota (docs).

  Accede a la ruta `/src/assets/icons`, crea un nuevo archivo y asignale un nombre (tiene que ser el mismo nombre asignado al Enum del componente). 
  Finalmente, pega el código SVG y guarda.
